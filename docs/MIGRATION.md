# Переход со старых интеграций

## Со старого BackTraderQuik + отдельного QuikPy

1. Удалите локальные копии `QKStore.py`, `QKBroker.py`, `QKData.py`, `QuikPy.py`
   из каталога стратегии.
2. Установите пакет и импортируйте публичный API из `backtrader_quik`.
3. Создайте один `QKStore()` на процесс.
4. Выберите счёт явно: `store.getbroker(account_id=...)`.
5. Передавайте полное имя инструмента `CLASS.SEC`.
6. Проверьте размеры: акции — в бумагах, фьючерсы — в контрактах.

Старый подход с набором скопированных файлов не даёт воспроизводимой версии и
легко смешивает несовместимые QKBroker/QuikPy/Lua. В пакете они версионируются
вместе.

## С BacktraderQuikJunior

Lua-коннектор и практические особенности Junior сохранены, но Python-слой теперь
следует стандартному интерфейсу Backtrader Store/Data/Broker. Не переносите
прямые вызовы транзакций, если их можно выразить через `buy()`, `sell()` и
`cancel()`.

Особенно проверьте:

- `account_id` и `client_code`;
- код класса (`QJSIM`, `SPBFUT` и т. п.);
- единицы `size`;
- защитную ширину `slippage_steps` для срочной Market-заявки;
- обработчики `notify_data`, `notify_order`, `notify_trade`.
