# Установка и подключение QUIK

## 1. Python

```powershell
py -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install .
```

Проверьте импорт:

```powershell
python -c "import backtrader_quik as q; print(q.__version__)"
```

Ожидается `1.0.0a4`.

## 2. Lua-коннектор

1. Создайте отдельный каталог рядом с QUIK, например `C:\QuikLua`.
2. Скопируйте в него содержимое `QUIK/lua` и `QUIK/socket`, не меняя
   относительное расположение файлов.
3. В QUIK откройте **Сервисы → Lua-скрипты**, добавьте `QuikSharp.lua` и
   запустите его.
4. Убедитесь, что локальные порты слушаются:

```powershell
Get-NetTCPConnection -State Listen -LocalPort 34130,34131
```

Оба порта должен обслуживать один процесс QUIK. Windows Firewall не должен
блокировать локальное соединение `127.0.0.1`.

## 3. Первый безопасный запрос

```powershell
python .\Examples\list_accounts.py
```

Запишите нужный `account_id`, `firm_id`, `trade_account_id`, `client_code` и
доступные `class_codes`. Если у фондового счёта несколько кандидатов
`client_code`, передайте правильный код явно в `getbroker()`.

## 4. Данные до заявок

Сначала проверьте `get_param_ex()` для `LAST`, `STATUS`, `BID`, `OFFER`, затем
получение истории и live-свечей. Статус `DELAYED` означает, что feed запущен, но
новый закрытый бар ещё не получен. Он сам по себе не доказывает зависание.

## 5. Заявки

Первые проверки выполняйте в QUIK Junior минимальным размером. После каждого
сценария сверяйте:

- статус заявки в Backtrader;
- заявку и сделку в таблицах QUIK;
- позицию;
- отсутствие активных обычных и стоп-заявок.

Не переносите настройки `slippage_steps` с одного инструмента на другой без
оценки шага цены и допустимой защитной границы.

## Диагностика

- `ConnectionRefusedError`: Lua-скрипт не запущен или порт не слушается.
- `TimeoutError` на первом запросе: socket есть, но Lua не обработал запрос;
  перезапустите скрипт и проверьте его сообщения.
- история есть только на 1 минуте: DataSource заданного периода пуст в Junior;
  используйте доступный период и resample в Backtrader.
- нет `OnTrade`: 1.0.0a4 резервно опрашивает только принятую Market-заявку по
  точному биржевому номеру; Limit/Stop должны продолжать идти обычным callback.
