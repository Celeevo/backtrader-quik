> BackTraderQuik относятся к этой же кодовой базе.

на фондовом и срочном рынках. Финальная независимая проверка подтверждает:

- позиция по `SPBFUT.RIU6` равна нулю;

- активных стоп-заявок нет.

## Обнаруженные и исправленные несовместимости

### 1. Нулевая цена фондовой стоп-заявки

BackTraderQuik 1.0.0a3 формировал фондовую заявку `Stop` с `PRICE=0`. QUIK
Junior отклонял такую транзакцию. Тестовый брокер теперь задаёт положительную
защитную цену за уровнем `STOPPRICE`.

### 2. Потерянные callbacks свечей и сделок

QUIK Junior может не передать callback новой свечи или исполненной сделки.
Стенд резервно опрашивает только конкретный подписанный источник свечей или
точный биржевой номер принятой рыночной заявки. Восстановленные события
проходят через штатные механизмы устранения дублей BackTraderQuik.

### 3. Особое значение client_code на срочном рынке

В описании фьючерсного счёта QUIK Junior возвращает пустой `client_code`, но в
сделке записывает туда идентификатор торгового счёта `SPBFUT0003k`. Из-за этого
BackTraderQuik отбрасывал корректную сделку как относящуюся к другому счёту.
Стенд допускает такое представление только при точном совпадении торгового
счёта и идентификатора фирмы.

### 4. Защитная цена фьючерсной рыночной заявки

QUIK Junior может использовать поле `PRICE` рыночной фьючерсной заявки как
предельную защитную цену. При диапазоне в 10 шагов заявка была автоматически
снята без исполнения после движения рынка. Для тестового стенда установлен
диапазон по умолчанию 100 шагов. Неактивная заявка с неисполненным остатком
теперь корректно распознаётся как отменённая без длительного ожидания.

### 5. Ненадёжное поле STATUS

В QUIK Junior поле `STATUS` иногда остаётся пустым во время активных торгов.
Перед аварийным закрытием `Flatten` стенд дополнительно требует одновременно:

- ненулевые `LAST`, `BID` и `OFFER`;
- положительное число сделок;
- свежую минутную свечу;
- отсутствие явно неактивного статуса, например «назначена».

## Безопасность тестирования

- Для каждого теста использовался один биржевой лот или один фьючерсный
  контракт.
- Перед отправкой транзакций проверялись счёт, инструмент, исходная позиция и
  отсутствие активных заявок.
- После каждого прогона состояние сверялось непосредственно с QUIK.
- Две временно оставшиеся позиции `+1 RIU6`, возникшие во время диагностики,
  были закрыты отдельным однократным режимом `Flatten` без повторных заявок.
- Финальное состояние подтверждено: позиция 0, активных заявок нет.

## Сценарии, оставленные для контролируемого ручного тестирования

Не форсировались следующие сценарии:

- частичное исполнение — оно зависит от фактического встречного объёма;
- намеренный отказ по марже через заведомо завышенный рыночный объём;
- ручное изменение заявок в терминале во время работы программы;
- биржевая экспирация и внешние изменения состояния счёта.

Эти проверки требуют отдельного контролируемого сценария и не влияют на
успешный результат основной матрицы BackTraderQuik с QUIK Junior.
