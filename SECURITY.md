# Политика безопасности

Не создавайте публичный issue, если проблема раскрывает данные счёта, позволяет
отправить транзакцию не на тот счёт или обойти проверку инструмента. Передайте
владельцу репозитория приватное описание с минимальным воспроизведением.

Поддерживается последняя опубликованная alpha-версия. Секреты, пароли и токены
не должны храниться в репозитории. Любая стратегия обязана иметь независимые
лимиты риска и процедуру аварийного закрытия.
